# 更新日志

本文件记录高校运动会管理系统的主要版本变更。

## v0.4 - 测试版安全加固

发布日期：2026-06-30

### 新增

- 新增轻量级安全校验工具 `SecurityValidator`
  - 校验 ID 必须为正数
  - 校验比赛类别仅允许 `男子 / 女子 / 混合`
  - 校验性别仅允许 `男 / 女`
  - 校验状态仅允许 `0 / 1 / 2`
  - 校验取前N名、名次、成绩文本等表单输入
  - 过滤搜索关键词中的危险字符，如 `'`、`"`、`;`、`--`、`/* */`、`<script>` 等
- 新增业务异常 `BusinessException`
- 新增全局异常处理器 `GlobalExceptionHandler`
  - 统一处理业务异常和参数格式错误
  - 避免非法 URL 或参数导致 Whitelabel Error Page
  - API 请求异常返回 JSON 错误结果
- README 增加详细 API 接口说明与测试指导
- 新增安全加固实施计划文档：`docs/superpowers/plans/2026-06-30-security-hardening.md`

### 优化

- 比赛项目搜索、运动员搜索、报名搜索、个人成绩查询、项目成绩查询全部改为 POST 表单提交
- 搜索后地址栏不再显示 `keyword`、`eventId`、`deptId` 等查询参数
- 比赛项目删除、结束比赛、运动员删除、取消报名等危险操作全部改为 POST 表单提交
- 手动访问旧 GET 危险地址时不执行操作，仅提示必须通过页面按钮提交
- Controller 层增加参数合法性校验和资源存在性校验
- 成绩录入页面对非法项目编号增加友好错误处理
- 报表页面对非法项目编号和非法关键词增加友好错误处理
- API 接口增加参数校验，非法关键词返回错误 JSON

### 修复

- 修复 URL 手动修改为不存在 ID 时可能出现 Whitelabel Error Page 的问题
- 修复搜索关键词携带 SQL 注入特征字符时缺少统一拦截的问题
- 修复删除/结束/取消等操作可通过 GET URL 直接触发的安全隐患
- 修复搜索表单使用 GET 导致 URL 暴露查询参数的问题

### 测试建议

- 登录后访问 `/event/delete/1`，应重定向回项目列表并提示不允许直接修改 URL 执行
- 登录后访问 `/athlete/delete/1`，应重定向回运动员列表并提示不允许直接修改 URL 执行
- 登录后访问 `/registration/cancel/1`，应重定向回报名列表并提示不允许直接修改 URL 执行
- 访问 `/score/input/999999`，应显示友好错误提示，不出现 Whitelabel
- 搜索框输入 `' OR 1=1 --`，应提示关键词包含非法字符
- 使用 JMeter 测试 `/api/login`、`/api/events`、`/api/report/team`，需添加 HTTP Cookie Manager 维持登录 Session

## v0.3 - 测试版

发布日期：2026-06-30

### 新增

- 完成基础业务功能
  - 用户登录/登出
  - 仪表板统计
  - 比赛项目管理
  - 运动员管理
  - 报名管理
  - 成绩录入
  - 统计报表
- 比赛项目列表支持直接结束比赛
- 比赛项目列表支持点击项目行展开已报名人员
- 报名增加性别限制
  - 男子项目仅男运动员可报名
  - 女子项目仅女运动员可报名
  - 混合项目男女均可报名
- 添加初始成绩数据

### 修复

- 修复项目管理搜索不可用
- 修复项目添加提交路径错误
- 修复运动员列表变量名不匹配
- 修复报名列表变量名不匹配
- 修复报名列表学号字段显示错误
- 修复成绩录入选择项目按钮无反应
- 修复个人成绩报表字段名不匹配
- 修复中文乱码问题
